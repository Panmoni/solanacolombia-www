Febrero es cuando empezamos a construir.

No habrá clases tradicionales. No habrá slides. No habrá teoría de 2 horas antes de tocar código.

Construirás desde el primer día. Con otros builders. Con mentores disponibles. Con el objetivo simple de deployar algo cada viernes.

Esto es **Protocolo Origen: Bootcamp de Febrero**.

## El Formato

Este bootcamp funciona diferente a lo que probablemente conoces.

No es un curso grabado. No es un webinar donde escuchas pasivamente. Es un espacio de trabajo estructurado donde construyes en tiempo real.

### Cómo Funciona Cada Sesión:

**[00:00-00:10]** → Te damos el challenge del día  
**[00:10-01:40]** → Construyes (soporte disponible, pair programming opcional, debugging colectivo)  
**[01:40-02:00]** → Show & Tell (algunos participantes muestran lo que hicieron)

**Principios:**
- Construyes primero, entiendes después
- Todos deployean algo cada semana
- Si te atascas, preguntas. Si ya lo lograste, ayudas

## Calendario Completo de Febrero 2026

### Semana 1

**Jueves 5 de Febrero, 12:00 PM → X Space**
- Tema: "De la Idea al Código: Ejercicio de Ideación Web3"
- Un ejercicio guiado para identificar qué construir
- No necesitas hablar. Escuchas, reflexionas y tomas notas

**Viernes 6 de Febrero, 4:00 PM → Build Lab #7: "Hello Solana"**

**Challenge:**
```
Deployar tu primer programa en Solana.
Un "Hello World" en la blockchain.

Tienes 90 minutos.
```

**Qué cubriremos:**
- Setup de ambiente de desarrollo
- Deploy a Devnet
- Conceptos básicos: cuentas e instrucciones
- Uso del CLI de Solana

**Resultado:**
- Tu primer programa deployado
- Una transacción visible en Solscan

### Semana 2

**Jueves 12 de Febrero, 12:00 PM → X Space**
- Tema: "Validation Day: Comparte Tu Idea"
- Los participantes presentan sus ideas
- Feedback del grupo y mentores

**Viernes 13 de Febrero, 4:00 PM → Build Lab #8: "Build a Counter"**

**Challenge:**
```
Construir un programa que guarda estado.
Un contador: incrementa, decrementa, reset.

Opcional: Agregar un timer (solo incrementa cada 10 segundos).
```

**Qué cubriremos:**
- Guardar datos en la blockchain
- Accounts y ownership
- Instrucciones con lógica condicional

**Formato:**
- Mob programming en grupos de 3-4
- Un grupo comparte pantalla, los demás guían
- Rotación de quien escribe código cada 15 minutos

### Semana 3

**Jueves 19 de Febrero, 12:00 PM → X Space**
- Tema: "Builders que Han Ganado Hackathons"
- Invitado especial: Un ganador previo de hackathon Solana
- Q&A

**Viernes 20 de Febrero, 4:00 PM → Build Lab #9: "Token Time"**

**Challenge:**
```
Crear un token SPL que represente algo real.

No genéricos. Algo relacionado con tu proyecto:
- Token de membresía
- Puntos de fidelidad  
- Representación de un asset

Deploy en Devnet. 90 minutos.
```

**Qué cubriremos:**
- Creación de tokens SPL
- Metadatos (nombre, símbolo, imagen)
- Permisos (mint authority, freeze authority)
- Casos de uso prácticos

### Semana 4

**Jueves 26 de Febrero, 12:00 PM → X Space**
- Tema: "Formación de Equipos"
- Presenta tu proyecto
- Match-making: dev + design + business
- Deadline para formar equipos

**Viernes 27 de Febrero, 4:00 PM → Build Lab #10: "Frankenstein Build"**

**Challenge:**
```
Sin template.

Combinar lo aprendido:
1. Programa con estado
2. Token SPL
3. Lógica que conecte ambos

Ejemplos:
- Check-in que da tokens
- Sistema de tareas con recompensas
- Marketplace simple

90 minutos. Usa recursos externos si los necesitas.
```

**Qué cubriremos:**
- Integración de conceptos
- Lectura de documentación
- Debugging independiente

**Show & Tell Final:**
- Todos muestran su trabajo (funcione o no)
- El grupo da feedback

## Requisitos

### Técnicos:
- Laptop con mínimo 8GB RAM
- Conexión a internet estable

### Tiempo:
- **Obligatorio:** Viernes 4-6 PM (Build Labs)
- **Recomendado:** Jueves 12-1 PM (X Spaces)
- **Autodidacta:** 5-10 horas/semana entre sesiones

### Mentalidad:
- No esperes un curso pulido. Es un espacio de trabajo colaborativo
- No todo funcionará a la primera. El debugging es parte del proceso
- Los mentores guían, no resuelven todo por ti

### Qué Incluimos:
- Templates para empezar
- Mentores disponibles durante las sesiones
- Telegram para soporte
- Estructura para mantener momentum

## Después de Febrero

**Marzo:** Build Phase (8 semanas)  
**Abril:** Mentoría de pitch y demo videos  
**Mayo:** Colosseum Hackathon  
**Junio:** Demo Day local

Febrero es la base. El resto depende del momentum que construyas aquí.

## Los Números

- **4 viernes**: 6, 13, 20, 27 de febrero
- **4 jueves**: 5, 12, 19, 26 de febrero  
- **4 programas** para deployar
- **1 token** para crear
- **90 minutos** de build time cada viernes

## El Compromiso

Esto es sin costo.

Pero requiere que aparezcas, construyas y ayudes cuando puedas.

Si buscas:
- Clases donde alguien explica cada línea de código
- Un ambiente sin fricción donde todo funciona perfectamente
- Una experiencia pasiva

Este formato probablemente no te sirva.

Si estás dispuesto a:
- Experimentar y romper cosas
- Hacer preguntas específicas
- Ayudar a otros cuando ya lograste algo
- Deployar aunque tu código tenga bugs

Bienvenido.
